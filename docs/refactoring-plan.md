### 1.2 Отчёт о багах и проблемах

> Уровни серьёзности: **Blocker / Critical / Major / Minor / Cosmetic**.



#### Blocker

- [ ] **B-1. JS-краш на** `privacy.html` **ломает навигацию.**
  `script.js` выполняется на `privacy.html`, где отсутствует слайдер. `reviewsDots = document.getElementById('reviewsDots')` → `null`, затем в `buildDots()` (`script.js:422`) вызывается `reviewsDots.innerHTML = ''` → `TypeError: Cannot set properties of null`. Всё, что идёт после (`burger menu`, `smooth scroll`, `active nav on scroll`), не инициализируется. На мобильных `privacy.html` бургер-меню не открывается.
  *Воспроизведение:* открыть `privacy.html` на ширине <768px, нажать бургер → меню не открывается; консоль содержит TypeError.



#### Critical

- [ ] **C-1. Несовпадение** `lang` **и языка контента.**
  `<html lang="ru">` на обеих страницах, но весь текст, `aria-label`, `title`, `meta description` — на английском. Скринридеры озвучивают английский текст русским произношением. Нарушение WCAG 3.1.1.
- [ ] **C-2. Контент скрыт при отключённом JS.**
  AOS.css задаёт `[data-aos] { opacity: 0 }`. Класс-«спасатель» `.aos-fallback` добавляется только из JS. Без JS элементы остаются `opacity:0` → страница невидима. Нарушение WCAG 1.3.2 / 4.1.3 (прогрессивное улучшение отсутствует).
- [ ] **C-3. Фокусable-элементы внутри закрытых модалок.**
  `.modal`/`.success-modal` имеют `aria-hidden="true"` и `pointer-events:none`, но кнопки закрытия, инпуты, toggle-кнопки остаются в Tab-последовательности (`tabindex` не `-1`, нет `inert`). Пользователь может табнуться в невидимую форму. Нарушение WCAG 2.4.3.
- [ ] **C-4. Отсутствует focus trap в модалках.**
  После открытия фокус переносится на кнопку закрытия, но Tab уходит за пределы диалога в фон. Нарушение WAI-ARIA Dialog Pattern.
- [ ] **C-5. Фокус не возвращается на триггер.**
  После закрытия модалки/ success-модалки фокус пропадает (не возвращается на кнопку, которая открыла). Нарушение WCAG 2.4.3.
- [ ] **C-6. Нет видимого фокус-индикатора у CTA-инпута.**
  `.cta__input { border: none; outline: none }`, а `:focus` меняет только цвет placeholder. Текстовое поле не показывает, что получило фокус. Нарушение WCAG 2.4.7.



#### Major

- [ ] **M-1. Клик по ссылке «Privacy Policy» внутри** `<label>` **чекбокса toggles чекбокс.**
  `modal__link` (`privacy.html`) лежит внутри `label.modal__checkbox-label`. Клик по ссылке одновременно активирует label и меняет состояние чекбокса. UX-баг и риск случайного согласия/отзыва.
- [ ] **M-2. Фокус-индикатор кастомного чекбокса обрезается.**
  `.modal__checkbox-label { overflow: hidden }`, а `outline` с `outline-offset: 2px` рисуется на `.modal__checkbox-custom` внутри label → outline клиппится. Нарушение WCAG 2.4.7.
- [ ] **M-3. Отсутствуют focus-стили для ряда ссылок.**
  Нет `:focus-visible` для `.header__logo`, `.modal__link`, `.privacy__link`, `.footer`-ссылок (если появятся). Нарушение WCAG 2.4.7.
- [ ] **M-4. Некорректная ARIA-модель слайдера.**
  `.reviews__dots` = `role="tablist"`, точки = `role="tab"`, но карточки не имеют `role="tabpanel"`/`id`/`aria-controls`. Паттерн «tabs» применяется неверно; корректно — `role="group"` + кнопки пагинации или carousel-pattern с `aria-roledescription="carousel"`.
- [ ] **M-5. Оверлеи модалок —** `<div>` **с** `aria-label` **и click, но не кнопки.**
  `.modal__overlay`/`.success-modal__overlay` — кликабельные div'ы с `tabindex="-1"` и `aria-label="Close..."`. Для скринридеров это не интерактивные элементы. Нужен `role="button"` или `<button>`.
- [ ] **M-6. Нет skip-link «К основному содержимому».**
  WCAG 2.4.1 Bypass Blocks. Клавиатурный пользователь вынужден таббиться через всю шапку.
- [ ] **M-7. Отсутствует favicon и social meta.**
  Нет `<link rel="icon">`, Open Graph, `theme-color`. Влияет на UX/SEO/шеры.
- [ ] **M-8.** `console.log` **в продакшен-коде.**
  `script.js:210` (`Request submitted`), `script.js:390` (`CTA form submitted`). Логи попадут в консоль браузера клиента.



#### Minor

- [ ] **m-1. Переполнение слайдера на ≥768px на ~6px.**
  `.review-card` в `@media (min-width: 768px)` имеет `flex: 0 0 calc(50% - 12px)`, а `gap: 30px`. Две карточки + gap = `100% + 6px` → 6px третьей карточки видны/клиппятся. Должно быть `calc(50% - 15px)` (как в базовом правиле).
- [ ] **m-2.** `lang`**/**`aria-label` **логотипа на русском среди английского UI.**
  `aria-label="RubyHome — главная страница"` — единственный русский aria-label, остальные английские. Несогласованность.
- [ ] **m-3. Избыточные ручные keydown-обработчики на** `<button>`**.**
  Для `.hero__button`, `.header__cta-btn`, `.property-card__btn`, `.modal__toggle`, `.header__burger` добавлены `keydown` на Enter/Space, хотя нативный `<button>` сам генерирует click. Мёртвый/избыточный код.
- [ ] **m-4.** `partners__container` **на мобиле — горизонтальный скролл без подсказки.**
  `overflow-x: auto` + `min-width: max-content` без scroll-hint и без ARIA-обёртки. Пользователь может не понять, что список прокручивается.
- [ ] **m-5.** `rel="noopener"` **без** `noreferrer` **на** `modal__link`**.**
  В `privacy.html` внешние ссылки используют `noopener noreferrer`, а в модалке — только `noopener`. Несогласованность best-practice.
- [ ] **m-6.** `will-change: transform` **на** `.reviews__track` **постоянно.**
  Указано всегда, а не только во время drag. Лишнее потребление памяти/GPU.
- [ ] **m-7. Success-модалка авто-закрывается через 3с без учёта скринридера.**
  Сообщение исчезает до того, как AT-пользователь его «прочитает». Нужен `aria-live`/ручное закрытие по таймауту только для зрячих.
- [ ] **m-8. Контраст** `.reviews__label` **(**`#fe753f` **на белом).**
  ~3:1 — на грани для 16px/600. Для UI-текста желательно ≥4.5:1.
- [ ] **m-9. Цвет введённого текста в CTA —** `rgba(3,10,27,0.50)`**.**
  Введённый (не placeholder) текст отображается полупрозрачным серым — низкий контраст. Стоит разделить цвет placeholder и цвет введённого текста.
- [ ] **m-10.** `meta name="keywords"` **устарел.**
  Игнорируется поисковиками. Можно оставить, но не несёт пользы.



#### Cosmetic / Code quality

- [ ] **q-1.** `script.js` **— один файл 590 строк, смешаны все домены.**
  AOS, modal, validation, CTA, slider, burger, scroll, nav. Нарушение SRP, трудно поддерживать.
- [ ] **q-2. Дублирование модальной разметки в** `index.html` **и** `privacy.html`**.**
  Нарушение DRY. Любое правка требует синхронизации двух файлов.
- [ ] **q-3. Смешение mobile-first и desktop-first в** `style.css`**.**
  Часть блоков (`.cta__form`, `.cta__inner`) задаёт десктопные значения в базе и переопределяет на мобиле через `max-width:767px`. Нарушает заявленный mobile-first.
- [ ] **q-4. «Магические числа» в JS.**
  `GAP=30`, пороги `60`/`50`, `86px` в CSS-брейкпоинте шапки, `100`/`50` смещения скролла. Стоит вынести в константы.
- [ ] **q-5.** `showError`**/**`clearError` **возвращают boolean, которое не используется.**
  Лишний контракт.
- [ ] **q-6. Имена файлов** `Rectangle 122.png`**,** `Rectangle 122 (1).png` **…**
  Пробелы и скобки в путях; URL приходится кодировать. Лучше переименовать (с обновлением ссылок).
- [ ] **q-7. Незакодированные пути в** `src` **(**`assets/img/Rectangle 122.png`**).**
  Работает, но не best-practice.

## ЧАСТЬ 2. План рефакторинга и оптимизации (`<Senior Fullstack Developer>`)

### 2.1 План рефторинга и оптимизации кода



#### 2.1.1 Разделение `script.js` на модули

- [ ] Создать каталог `js/` и разбить логику по доменам:
  - [ ] `js/aos-init.js` — инициализация AOS + fallback.
  - [ ] `js/modal.js` — открытие/закрытие модалок, focus trap, возврат фокуса.
  - [ ] `js/form-validation.js` — валидаторы (`validateName/Phone/Email`), `showError`/`clearError`.
  - [ ] `js/request-form.js` — форма заявки в модалке + toggle Buy/Rent/Sell.
  - [ ] `js/cta-form.js` — CTA-форма и success-toast.
  - [ ] `js/reviews-slider.js` — слайдер (точки/drag/swipe/keyboard), **с guard на отсутствие элементов**.
  - [ ] `js/navigation.js` — бургер, smooth scroll, active-link, skip-link.
  - [ ] `js/main.js` — точка входа, импорт/вызов модулей по порядку, единый `DOMContentLoaded`.
- [ ] Каждый модуль: early-return если нужных элементов нет на странице (решает B-1).
- [ ] Сохранить итоговый бандл как `script.js` (сборка) **или** подключить несколько `<script type="module">` — без сборщика предпочтителен вариант с единым `script.js`, разбитым на IIFE-секции с guards, чтобы не менять разметку подключения. **Решение:** оставить один `script.js`, но реорганизовать в IIFE-модули с защитой `if (!rootEl) return;` в каждом блоке (минимальный риск, ничего не ломает).



#### 2.1.2 Устранение дублирования HTML

- [ ] Не дублировать модалку руками: оставить как есть в обеих страницах (статика без сборщика), но **вынести общую модалку/шапку/футер в согласованную структуру** через единообразную разметку. Без шаблонизатора — оставить дубль, но добавить комментарий-маркер `<!-- SHARED: modal markup — keep in sync -->` в обоих файлах. (Альтернатива: ввести минимальный HTML-include через JS — **отклонено**, ломает no-JS.)
- [ ] Привести `aria-label` и `lang` к единому языку (см. 2.5).



#### 2.1.3 Принципы и инварианты рефакторинга

- [ ] DRY: общие валидаторы используются и в форме заявки, и в CTA.
- [ ] SRP: одна функция — одна ответственность.
- [ ] Early returns, `const` arrow-functions, `handle*`-нейминг для обработчиков (по правилам проекта).
- [ ] Все обращения к DOM предварять проверкой существования элемента.
- [ ] Вынести «магические числа» в константы (`GAP`, `DRAG_THRESHOLD_MOUSE`, `DRAG_THRESHOLD_TOUCH`, `SCROLL_OFFSET`).
- [ ] Не трогать селекторы и значения в `style.css`, влияющие на визуал (допускается только добавлять `:focus-visible`-правила и `inert`-совместимые стили).
